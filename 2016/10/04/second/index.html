如何在github上搭建个人博客教程

写在前面

首先感谢大家参加助力课堂，参加我的这门课程，其实课上我也说过了，我自己也是刚刚接触github pages没多久，觉得有这么一个简易而且免费的平台，不推荐给大家实在可惜，加上上学期也在助力课堂当过讲师（虽然第一次参加比较失败…听课人数个位数），但是这并不能阻挡我！

为了世界的和平，我又回来了！希望大家能真正学到一些知识，可以亲手搭出一个免费的博客，自豪感是满满的哦~

关于github pages

github pages官方介绍

Websites for you and your projects.
Hosted directly from your GitHub repository.
Just edit, push, and your changes are live.

从这里的官方介绍可以看出来，github为了贯彻自己的宗旨，为所有用户都提供了这样一个免费的空间，你可以do whatever you want,除了用它来搭建个人博客，也可以当作你项目的主页，甚至是做成一个小型的企业网站，这些都是被允许的。

但是Github Pages只能存放静态页面，不能运行后台代码，不能动态生成文件，也不能有数据库。

你看到的“博客引擎”其实是页面生成器，即把你写的模版等动态代码转成静态页面，然后才能放到Github上。所以，github pages并不是万能的。

敲黑板！！！下面这段很关键！
由于github pages本质上还是一个repository，所以它是可以被整个clone的！！所以用来当企业网站还是不太合适，但同时大家也可以利用这个特性，clone下来自己喜欢的博客来看他们是怎么搭建的~

下面我们正式开始！

环境配置

在这里我就只讲解windows下的环境配置，至于其他系统的配置方法都大同小异，大家参照着下面的来吧~

1.配置Node.js

首先下载安装包，希望大家自行去官网下载最新版的安装包，这也是一项必不可少的技能哦~一定注意32位和64位要选择对，如果不知道自己电脑是32位还是64位的右键–>我的电脑–>属性 就可以看到了。如果嫌麻烦的话就直接点下面的链接下载吧！

  • Windows Installer 32-bit
  • Windows Installer 64-bit

下载完成后双击打开安装包，如图所示：

按照默认的设置，一路next下去就可以很快安装好了。安装好之后，我们按住win+R打开运行，输入cmd，
打开命令行窗口，分别输入：
node -v
npm -v

到此Node.js就安装好了

至此git的安装就结束了

github账号的配置

1.账号注册

首先，前往github官网，注册一个github账号。

注册成功之后便可以登录自己的github账号了，关于github的其他应用，有机会我也会与大家分享的~

2.新建仓库

点击右下方的New repository新建一个仓库，

在下一个界面中，填入你的仓库名称,注意！！！一定要像图中的格式一样，为XXXXX.github.io，其中XXXXX代表你的Username，且一个github账户只能有一个io仓库来作为github pages。由于我已经有一个同名仓库了，所以系统会提示我already exists

在Description里你可以填入一些描述，当然不填也可以，在下面的Public和Private选项中你可以选择公开或私密，以防止以后有人clone你的主页。

正确创建之后，你将会看到如下界面：

3.开启gh-pages功能

点击界面右侧的Settings，你将会打开这个库的setting页面，向下拖动，直到看见GitHub Pages，如图：点击Automatic page generator，Github将会自动替你创建出一个gh-pages的页面。
如果你的配置没有问题，那么大约15分钟之后，yourname.github.io这个网址就可以正常访问了~
如果yourname.github.io已经可以正常访问了，那么Github的配置就完成了。

配置Hexo

1.什么是hexo？

这是hexo的主页：hexo主页，

官方对hexo的解释是：A fast, simple & powerful blog framework, powered by Node.js.

其实hexo就是一个作业本，但是它的款式却多种多样（主题），它可以帮你生成你的每一篇日志，并且可以很简单的帮你发布~下面我们开始配置它。

2.安装Hexo

在本地磁盘的一个合适的位置新建一个文件夹，它就是你博客的本地映射了，命名随意，最好不要用中文。然后双击进入这个文件夹，按住shift+鼠标右键，选择“在此处打开命令行”，当然你也可以用win+R，输入cmd，然后一步一步的cd到这个文件夹的目录，但是需要耗费很多时间啊！这也是一个省时省力的小技巧~
接下来，输入：
npm install hexo-cli -g

然后你将会看到:可能你会看到一个WARN，但是不用担心，这不会影响你的正常使用。然后输入

npm install hexo --save

然后你会看到命令行窗口刷了一大堆白字，下面我们来看一看Hexo是不是已经安装好了。在命令行中输入：
hexo -v

如果你看到了如图文字，则说明已经安装成功了。

3.初始化Hexo

接着上面的操作，输入：
hexo init

如图：然后输入：
npm install

之后npm将会自动安装你需要的组件，只需要等待npm操作即可。
首次体验Hexo
继续操作，同样是在命令行中，输入：
hexo g

如图：然后输入：
hexo s

然后会提示：
INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

在浏览器中打开http://localhost:4000/
，你将会看到：到目前为止，Hexo在本地的配置已经全都结束了。

首先我们可以修改全局配置文件，右键文件_config.yml，用你的文本编辑器打开它，进行下面的这些选项的设置。
此段落引用自Hexo官方文档

这里的配置大多不用现在就更改，等到后面我们套用主题时再修改也不迟~先进行下一步吧！

2.hexo部署到github pages

这是目前为止非常关键的一步，也是重点和难点，大家如果遇到困难了一定要多问哦！

首先，你需要为自己配置身份信息，打开命令行，然后输入：
git config --global user.name "yourname"
git config --global user.email "youremail"
其中，yourname和youremail分别为你github的username和邮箱

接下来使用git方式进行部署，执行
npm install hexo-deployer-git --save
来安装所需的插件

同样在_config.yml文件中，找到Deployment，然后按照如下修改：
deploy:
type: git
repo: git@github.com:yourname/yourname.github.io.git
branch: master

注意用你的Username替换yourname

然后在项目文件夹内打开命令行，输入：
hexo d

随后按照提示，分别输入自己的Github账号用户名和密码，开始上传。然后通过http://yourname.github.io/来访问自己刚刚上传的网站。
至此，我们的个人主页搭建就已经初具规模了~~
